谈谈闭包

May 14, 2022 · 9 min read
Nikki
a coder

”闭包是什么？“

作为一道经典面试题，80%的人会直接回答：闭包就是一个函数里面嵌套一个函数，然后返回一个函数。而通过这段时间的学习，我可能会这样回答：

当我们通过调用一个外部函数返回一个内部函数后，即使该外部函数已经执行结束了，但是内部函数引用外部函数的变量依然保存在内存中，我们就把这些变量的集合称为闭包。

Q1：闭包在代码中的表现？什么叫内部函数引用外部函数的变量并返回一个函数？​
function outer() {
  let str = 'aaa';
  return function () {
    console.log(str); //内部函数引用了外部函数outer的变量str，并return这个内部函数
  };
}
var fn = outer(); //aaa
Q2：闭包为什么会出现？​

这个问题要从作用域和作用域链讲起。

ES6 之前的作用域分为全局作用域和函数作用域，在控制语句中用 var 定义的变量可以被外部访问，函数内部可以访问外部用 var 定义的全局变量。而 ES6 后，出现了由 let 和 const 定义的块级作用域，块级作用域内定义的变量外部是不可以访问的。

而作用域链则是作用域查找变量的链条，如果在当前作用域没有找到，就会去上级作用域查找，直到全局。

我们有时需要在函数外部读取函数内部的局部变量，并且需要变量持久保存，可是因为作用域的问题，我们很难读取到，并且由于垃圾回收机制，函数执行完没有引用后，变量所占据的内存就会被销毁。为了解决以上问题，就出现了闭包，可能有朋友会想，如果想在函数外部读取内部的局部变量，直接在函数中返回这个变量就行，可是，我们假设下面的场景:

1.为了安全考虑，我们只想让内部函数操纵外部函数中的变量，然后在外部拿到，保证变量的安全

2.我们需要在内存中维护一个变量，拿到外部函数的返回值后，想在连续在返回值的基础上做操作，比如

function add() {
  var i = 1;
  console.log(i++);
}
add(); //2
add(); //2
//我们希望第二个add函数能在第一个add函数执行过后的值的基础上进行累加，也就是值为3

此时，就可以使用闭包

function add() {
  var i = 1;
  return function b() {
    i++;
    console.log(i);
  };
}
var a = add();
a(); //2
a(); //3

在上面这个例子中，add 函数执行完并返回后，闭包使得垃圾回收机制不会收回 add 函数所占用的资源，因为 add 函数内部函数 b 的执行需要依赖 add 中的变量，因此变量 i 会一直存在于内存中，也就实现了累加的功能；

若 b 函数没有被 add 函数返回，而只是形成了嵌套和引用变量的关系，那么在 add 执行完之后，add 和 b 都会被垃圾回收机制回收。

同样，通过上面的例子，我们也可以得出形成闭包的条件

  • 函数嵌套函数
  • 内部函数引用外部函数的变量
  • 内部函数作为返回值返回
  • 外部函数被调用
通过一道经典的例题更加了解闭包的应用​
for (var i = 0; i <= 3; i++) {
    setTimeout(function () {
      console.log(i);//4444
    }, 1000);
  }
}

在上面这个场景中，我们想象中的场景是打印出 0,1,2,3。而打印出来的却是 4 个 4，这是由于 i 是全局变量，js 执行环境是单线程，setTimeout 是宏任务，轮到 4 个 setTimeout 执行的时候，for 循环已经执行完毕，i 的值已经变成了 4，所以打印出的 i 都是循环后的 4。

如何改造它让它打印出 0,1,2,3?​
//法一 使用立即执行函数形成局部作用域，立即执行函数执行时，会用每一轮得到的i值，赋给形参x进行运算
for (var i = 0; i <= 3; i++) {
    (function (x) {
      setTimeout(function () {
      console.log(x);
        }, 1000);
    })(i)
}
//上面的执行步骤
1.创建全局执行上下文for循环，推入执行栈
2.继续往下执行，此时i=0，创建立即执行函数上下文(function(x){}(i))
3.浏览器处理任务队列，此时x为0
4.立即执行函数执行完之后，从栈中弹出
5.回到全局执行上下文，进入第二次for循环i=1
6.以此类推...
7.直到全局执行上下文执行完出栈，打印出0,1,2,3
//法二 使用ES6新特性let
for (let i = 0; i <= 3; i++) {
    setTimeout(function () {
      console.log(i);//4444
    }, 1000);
  }
}
而在上文的法二中，通过使用 let 就解决了这个问题，为什么呢？​

要解答这个问题，我们首先要明白，每个函数都有自己的执行上下文，执行上下文中包含变量环境和词法环境。用 var 声明的变量，会存在于变量环境，用 let 和 const 声明的变量，会存在于词法环境。上面说到，作用域链是作用域查找变量的链条。而作用域链又是通过词法作用域来确定的，也就是在编译的时候，就根据函数所处的位置确定好了作用域链。执行上下文中同样还包含一个 outer 的指针，指向上级函数的执行上下文，只有在函数在位置上进行嵌套的时候，outer 才会生效，像下面这样

//此时b的outer指a
function a() {
    let b = 1;
    function b() {
        console.log(b)
    }
}
a()//输出1

//此时b的outer不指a
function b() {
    console.log(b);
}
function a() {
    let b = 1;
    b();
}
a()//b is not defined

而为什么两个函数的执行结果不同呢？

这是由于变量的查找顺序是：当前函数执行上下文(词法环境->变量环境)->outer 指向的上级函数执行上下文->全局执行上下文

词法作用域的规则就是内部函数可以访问外部函数的变量，利用这个规则，也就形成了我们今天所讲的闭包，写到这儿，我们再回顾一下闭包的概念：

当我们调用一个外部函数，他返回一个内部函数，这时即使外部函数已经执行完毕，但内部函数引用外部函数的变量依然保存在内存中，我们把这些变量的集合叫做闭包。

所以 let 解决了这个问题同样也是利用了词法作用域，在当前的词法环境中就找到了对应的值，最后打印的才是 0,1,2,3.
